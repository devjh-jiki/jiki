Introduction

This is a self-contained documentation template driven entirely by design-system tokens. Everything you see — colors, spacing, type, radius — resolves from tokens.css. Theme the docs by editing that one file; the layout never changes.

●
Token-driven. The sidebar, content, and table of contents all read from the semantic token layer. Swap --color-accent and the whole site re-themes.

Installation

Copy this template into your project and replace the placeholder content. No build step, no dependencies — open index.html in a browser.

cp -r docs-template/ my-project/docs/
# edit tokens.css, then fill in content

Quick start

The three-column shell gives you section navigation on the left, readable prose in the center (capped at a comfortable line length), and an on-page table of contents on the right that tracks your scroll position.

// code blocks use a separate GitHub/Primer syntax palette
import { createDocs } from './docs';

const site = createDocs({ theme: 'tokens.css', port: 3000 });
  • Left sidebar groups pages by section.
  • Center column holds the markdown-style prose.
  • Right TOC highlights the current heading.

Configuration

All layout dimensions are tokens too:

TokenDefaultControls
--sidebar-w260pxleft nav width
--content-max760pxreadable line length
--toc-w220pxright TOC width
▲
Keep prose readable. Don't widen --content-max past ~75 characters per line — long measures hurt reading.

Theming

Light mode lives in :root; dark mode overrides the same semantic tokens under [data-theme="dark"]. The toggle in the top bar flips the attribute and persists the choice.

API

Use a reference layout for lookup-oriented content: signature, parameters table, returns, example. The prose styles above cover all of it.
