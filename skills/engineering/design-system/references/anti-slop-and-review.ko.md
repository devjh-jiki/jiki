# Anti-slop & 디자인 리뷰

두 가지: UI를 AI 생성처럼 보이게 하는 구체적 패턴(과 대안), 그리고 UI를 채점하는 루브릭. **구현** 전에(티를 안 내려고), **리뷰** 중에(감이 아니라 기준으로 채점하려고) 읽는다.

## Anti-slop 금지 목록

이것들은 AI가 UI를 만들 때의 기본 출력이다. 각각이 티이고, 각각 더 나은 수가 있다. 이걸 피하는 게 진짜 제품과 템플릿을 가르는 일의 대부분이다.

| 티 (피하기) | 대신 |
|-------------|------|
| 보라→파랑 그래디언트를 기본 브랜드 룩으로 | 중립 surface 위 자신 있는 악센트 하나; 그래디언트는 브랜드가 정당화할 때만 |
| 이모지를 아이콘으로(버튼·기능에 🚀 ✨ 🔥) | 실제 아이콘 세트(일관된 굵기/그리드); 없으면 깔끔한 텍스트 라벨 |
| 네온-거의검정(`#0D1117` + 채도 높은 악센트)을 "모던 다크"로 | 시맨틱 층으로 고려된 다크 테마; 가라앉은 surface, 접근 가능한 악센트 |
| 아이콘-제목-텍스트의 대칭 3열 기능 그리드 | 콘텐츠에 맞게 레이아웃 변주; 비대칭과 진짜 계층이 디자인돼 보인다 |
| AI 생성 SVG 일러스트 / 스톡 느낌 히어로 블롭 | 실제 에셋 전까지 깔끔한 플레이스홀더(회색 박스, 실제 사진, 또는 없음) |
| 일반 디스플레이 폰트(어디나 Inter/Roboto, 의도 없음) | 의도적 타입 페어링; 최소한 디스플레이 vs 본문 구분 |
| 미묘하게 다른 버튼 다섯 종 | primary 하나, secondary 하나, ghost 하나 — 변형은 의도적 |
| 모두 중앙 정렬, 같은 무게, 초점 없음 | 화면당 명확한 primary 액션과 단일 시각 초점 |
| 모든 것에 드롭섀도 | elevation을 레이어링 신호로, 아껴서 |
| 경쟁하는 브랜드색 무지개 | 악센트 하나를 아껴 쓰고, 중립이 나머지를 진다 |

## 행동 휴리스틱

피할 것만이 아니라, 일하는 법:

- **빈 캔버스가 아니라 맥락에서 시작한다.** 기존 시스템/브랜드/가까운 컴포넌트를 먼저 찾아 맞춘다. 진짜 없으면 스타일링 *전에* 짧은 브랜드/토큰 스펙을 쓴다 — 픽셀 단위로 즉흥 하지 말고.
- **하나의 완성품이 아니라 구분되는 방향 2–3개를 제시한다.** 초기엔 실제 선택지 몇 개("에디토리얼 / 웜-미니멀 / 펑셔널")를 보여 사용자가 운전하게, 완성품 하나를 수락/거부하게 하지 말고.
- **회색 박스가 나쁜 에셋을 이긴다.** 중립 플레이스홀더는 레이아웃을 정직하게 전달하고, 나쁜 AI 이미지는 "미완성이고 촌스러움"을 전달한다.
- **화면당 시그니처 디테일 하나에 120% 노력.** 어디나 일반적인 게 슬롭이다. 요소 하나 — 고려된 빈 상태, 좋은 호버, 실제 일러스트, 독특한 spacing — 를 골라 진짜 좋게. 그 의도적 디테일 하나가 장인 정신으로 읽힌다.
- **시스템 먼저, 채우지 마라.** 여백은 채울 빈틈이 아니라 기능이다. 절제가 자신감으로 읽힌다.
- **폴리시 패스에선 더하지 말고 다듬어라.** "뭘 더할까"가 아니라 "여기 있는 걸 어떻게 더 낫게"를 물어라. 경쟁하는 요소를 빼는 게 새것 더하기보다 나을 때가 많다.

## 리뷰 루브릭

각 차원을 0–10으로 채점. **7 미만 차원은 실패 — 구체적 문제와 수정을 명명하고, 끝내려고 통과시키지 마라.** 판정이 아니라 점수를 보고한다.

### 1. 토큰 & 일관성 (0–10)
- 모든 color·space·radius가 토큰으로 추적되나? `#hex`나 `13px` 매직넘버 없나?
- 컴포넌트 역할마다 하나(primary/secondary/ghost 버튼; 카드 베이스 하나)?
- 형제 요소가 같은 radius·spacing 리듬·타입 역할을 쓰나?
- *실패 신호:* 하드코딩 값, 회색 여섯 종, 일관 없는 모서리 radius.

### 2. 접근성 (0–10)
- 대비가 WCAG AA: 본문 **4.5:1**, 큰 텍스트·UI/그래픽 **3:1**?
- 터치 타겟 **≥44×44px**? 모바일 인풋 텍스트 **≥16px**?
- 포커스 상태 보이나? 인터랙티브 요소 키보드 접근? 이미지 alt?
- *실패 신호:* 흰 위 연회색 본문, 작은 탭 타겟, 안 보이는 포커스.

### 3. 시각적 계층 (0–10)
- 화면당 명확한 primary 액션 하나?
- size / weight / color / spacing / position이 분명한 읽기 순서를 만드나?
- 가장 중요한 걸 2초 안에 찾나?
- *실패 신호:* 모두 같은 무게, 경쟁하는 CTA 둘, 초점 없음.

### 4. Spacing & 리듬 (0–10)
- spacing이 스케일(4px 기반)을 따르고 형제 간 일관되나?
- 진짜 여백(섹션 모바일 ≥48px / 데스크톱 ≥64px)? 답답하지도 표류하지도 않나?
- 그리드 정렬; 관련된 건 묶고 무관한 건 분리?
- *실패 신호:* 임의 margin, 가장자리까지 꽉 찬 콘텐츠, 들쭉날쭉 gap.

### 5. Anti-slop / 장인 정신 (0–10)
- 금지 목록 티가 하나도 없나?
- 의도적이고 브랜드다운 디테일이 고노력으로 최소 하나?
- 일반 템플릿이 아니라 특정 제품처럼 보이나?
- *실패 신호:* 보라 그래디언트, 이모지 아이콘, 대칭 기능 그리드, 어디나 일반적.

**판정:** 모든 차원 ≥7이고 접근성 실패가 없을 때만 통과(접근성 실패는 다른 점수와 무관하게 블로킹).

## 흔한 실수 → 수정

| 실수 | 수정 |
|------|------|
| 하드코딩 `#333` / `padding: 13px` | 시맨틱 토큰 참조; 진짜 역할이 빠졌으면 스케일에 추가 |
| 본문이 연회색(흰 위 #aaa) | 4.5:1 충족 토큰으로 올림 |
| 한 화면에 primary 버튼 둘 | 하나를 secondary로 강등; 뷰당 primary 액션 하나 |
| 일회성 spacing 40개 | 4px 스케일로 수렴; 이탈값 삭제 |
| 이모지 기능 아이콘 | 일관된 아이콘 세트나 일반 라벨로 교체 |
| 모두 중앙, 초점 없음 | 계층 세우기; primary 액션에 시각 무게 |
| 있는데 새 컴포넌트 | 기존 컴포넌트 재사용/확장 |
